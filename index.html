全聯店點 · Google Maps · CSV workflow

把跑店名單轉成可執行的分段路線。

這不是展示頁，而是一張工作表：貼 API key、上傳 CSV、解析店點座標，再輸出 Google Maps 路線與 routes.txt。

01

路線輸入

先確認起終點與 CSV 欄位。API key 只在瀏覽器端使用，仍請限制網域與 API。

Routes API 單次最多 23 個中繼點。
Google Maps app 會依此分段。
偏好
無標題列；預設第 0 欄店名、第 2 欄地址。
02

輸出

路線連結、處理紀錄與下載按鈕會在這裡出現。

尚未產生路線。完成左側欄位後，上傳 CSV 並開始產生。

03

限制與準備

保留實際工作所需資訊，不做額外裝飾。

API

  • API Key 需設定 HTTP 來源限制與 API 限制。
  • 需允許 Routes API、Places API、Geocoding API。
  • Routes API 單次最多 25 個點，含起點與終點，所以最多 23 個中繼點。
  • 機車模式部分地區可能尚未支援，失敗時會自動改用汽車模式。

CSV

  • 目前以全聯福利中心店點格式為主。
  • 若來源是 Excel，請先轉成 CSV。
  • 保留需要跑的店點；如果欄位不同，調整店名與地址欄位索引。
